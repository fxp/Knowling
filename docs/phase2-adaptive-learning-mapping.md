# 第二阶段规划：Adaptive Learning 产品 ↔ Knowling 边界划分

> 输入：用户 `/goal`——做一个真正的 Adaptive / Agentic Learning 产品（初期初中数学）：
> 照片解析试卷 → 推断知识点掌握 → 图谱点亮 → 自适应测题定位进度 → 生成学习计划
> → 逐知识点学习(互动+语音)+Quiz，答错则重教直到答对 → 点亮 → 跨次动态调整计划。
>
> 本文回答两件事：**哪些融入 Knowling（仍单 KP）**，**哪些整合为第二阶段新建的编排层**。

---

## 0. 核心结论

`/goal` 是一个**完整自适应学习系统**；Knowling 是其中**最原子的一层**——「把单个知识点学透」的学习单元工厂。

> goal 里的「知识点图谱点亮、自适应测题、计划生成、跨次调整」**全部属于编排层**，
> 是 ROADMAP 已 ✂︎ 砍掉的 P3（拆解+导航）方向的超集。**绝不塞进 Knowling 内核**，
> 否则破坏 single-KP focus 与自包含/可质检的护城河。
>
> Knowling 在本阶段只做两件事：① 把单 KP 学习单元做强（答错重教循环）；
> ② 长出干净的**接缝**（接口化 + 掌握度信号输出），供编排层调用。

---

## 1. 分层架构

```
┌──────────────────────────────────────────────────────────────┐
│ L1 诊断层   照片OCR → 对/错题提取 → 知识点掌握推断(+原因)        │  第二阶段新建
│             → 知识点图谱红绿灯点亮(带强度)                       │
├──────────────────────────────────────────────────────────────┤
│ L2 规划层   自适应测题(最小题量定位进度) → 学习计划(频次/内容)    │  第二阶段新建
│             → 每次学习的 KP 路径排序 → 跨次/频次动态调整          │
├──────────────────────────────────────────────────────────────┤
│ L3 学习单元 单KP: 互动教学内容 + 教学语音 + Quiz                 │  Knowling（已有，小扩）
│             + 答错重教循环(改得更易懂→直到答对)                  │
├──────────────────────────────────────────────────────────────┤
│ L0 课程数据 初中数学大纲 + 知识点图谱(节点 + 前置/包含边)         │  第二阶段新建（数据资产）
└──────────────────────────────────────────────────────────────┘

接缝 (the seam):
  L1/L2  ──(调用)──►  Knowling P4 接口 (gen / refine / quiz-eval)
  Knowling  ──(返回 每KP掌握度信号)──►  L1 图谱点亮
  L0 图谱节点  ──(KP坐标: 年级/章/节/id)──►  Knowling KnowledgePoint
```

## 3. 融入 Knowling 的部分（本阶段配合做，严格单 KP）

按价值/优先级：

### ① 答错重教循环（信号驱动 refine）— 最高价值
- **是什么**：输入「Quiz 得分 + 错因/错项」→ 自动重生成「更易懂」的同一 KP 学习卡（降难度/换讲法/加铺垫），循环直到 Quiz 达标。
- **为什么是 Knowling**：纯单 KP；是 goal 第 9 步的内核。
- **复用**：现有 `capabilities/refine.py`（当前是「太难了」对话驱动）信号驱动化；
  复用 `qa/loop.py` 的「只重编译失败块 + 建议注入」机制。
- **新增契约**：`refine_on_quiz_result(spec, quiz_result, reason) → new spec/html`。

### ② 每 KP 掌握度信号输出 — 集成接口（contract）
- **是什么**：学习单元结束时返回结构化结果：Quiz 得分 → 掌握/未掌握；可带**四轴**
  （memory / comprehension / structure / application，单调不降）。
- **为什么**：这是 L1 点亮图谱的**唯一契约**。上一轮 `borrow-analysis.md` 的「四轴掌握度」
  在此从「锦上添花」升级为**集成接口**。
- **落点**：`MasteryResult` 数据结构 + Knowling 单元出口。

### ③ 讲解级教学语音（TTS 旁白）
- **是什么**：把「教学语音」从 audio 模态块扩成可对整卡讲解的旁白轨（分步讲解）。
- **落点**：扩 `blocks/audio.py` 或新增 narration 能力；仍自包含（内联音频/或 Web Speech 降级）。

### ④ KnowledgePoint 带大纲坐标 + 稳定 id
- **是什么**：`KnowledgePoint` 增加 `curriculum`(年级/章/节) + 稳定 `kp_id`。
- **为什么**：让每个 Knowling 单元可被 L0 图谱节点寻址（图谱节点 ↔ Knowling 单元 1:1）。
- **落点**：`schema/models.py` 小改，向后兼容。

### ⑤ P4 接口化（MCP Tool / Server API）— 接缝，必须前置
- **是什么**：`knowling_generate / refine / quiz_eval` 暴露为 MCP Tool 与 `POST /v1/knowling/*`。
- **为什么**：L1/L2 编排层调用 Knowling 的**唯一入口**；ROADMAP P4 已规划，第二阶段须前置。

### ⑥（可选）Feynman 块
- 输出式/费曼学习，单 KP 内丰富教学，与本规划不冲突；非关键路径。

## 4. 整合到第二阶段（新建编排层，**不进 Knowling**）

这是产品的「Agent 大脑」，独立于 Knowling 之上，通过接缝调用 L3：

- **L0 课程数据**：初中数学大纲 + 知识点图谱（节点、前置/包含边、KP 坐标）。
- **L1 诊断**：onboarding 画像 · 试卷照片 OCR · 对/错题提取 · 题→KP 映射 · 掌握推断(+原因) · 图谱红绿灯点亮(强度)。
- **L2 规划**：自适应测题(最小题量定位进度) · 学习计划生成 · 每次 KP 路径排序 · 跨次/频次动态调整。
- **状态**：跨 KP / 跨会话的学习者进度与图谱点亮状态存储（学习档案）。
- **会话 runner**：onboarding → 提交试卷 → 看图谱 → 测题 → 出计划 → 「开始学习」逐 KP 走 → 当天结束 → 调整。

> 注意：L1 的「知识点图谱」与 Knowling 卡内的 `concept_graph` 块**不是一回事**——
> 前者是跨 KP 导航/状态（编排层），后者只是单卡内的学习辅助图。勿合并。

---

## 5. 建议的推进顺序

1. **先定接缝契约**：`KnowledgePoint.curriculum/kp_id` + `MasteryResult` + P4 接口签名（让两层能对接）。
2. **融入 Knowling ①②**：答错重教循环 + 掌握度信号输出（L3 自洽闭环：学→测→重教→出信号）。
3. **第二阶段起步**：L0 大纲图谱(初中数学) + L2 一个最小「计划→路径→调 Knowling→收信号→点亮」闭环（先跳过照片 OCR，用手动选 KP 验证编排）。
4. **再补 L1 照片诊断 / 自适应测题**（独立、可后置，不阻塞主闭环）。
5. ③ 教学语音 / ⑥ Feynman 按需。

---

## 6. 一句话总结

> 把 `/goal` 拆成四层：**L0 课程数据 / L1 诊断 / L2 规划 = 第二阶段新建的 Agent 编排层**（含 goal 里全部图谱、测题、计划逻辑）；**L3 单 KP 学习单元 = Knowling**。
> 本阶段 Knowling 只需长出「答错重教循环 + 掌握度信号输出 + 接口化」这三处接缝，
> 其余一律在 Knowling 之上的新层实现，保住 single-KP focus。
